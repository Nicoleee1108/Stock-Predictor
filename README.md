# Stock-Predictor — Market Regime Detection with a Gaussian HMM

> CS179 (Probabilistic Graphical Models) course project.
> We fit a **Gaussian Hidden Markov Model (HMM)** to daily stock-market data to
> discover hidden "market regimes" (calm bull markets, choppy periods, crises),
> then measure how well it works and how its performance changes as we change the
> model and the data.

This README is written to be read **top-to-bottom by a beginner**. The first half
explains *what an HMM is and why we use one*; the second half is *what we built,
how to run it, and what we found*.

## 1. What problem are we solving?

A stock price doesn't behave the same way all the time. Sometimes the market is in
a **calm uptrend** (small daily moves, gentle gains). Sometimes it's in a
**crisis** (huge swings, sharp losses). These different "moods" are called
**market regimes**.

The catch: **nobody hands us a label saying "today is a crisis."** We only see the
prices. We want to *infer* the hidden regime each day, using only the observable
price behavior.

This is exactly what a **Hidden Markov Model** is built for: there is a hidden
state that changes over time, and we only observe something *generated by* that
state. Our job is to work backwards from the observations to the hidden states —
**without any labels** (this is unsupervised learning).

We use the **S&P 500 ETF (SPY)** as the primary series because it represents the
whole market, so "bull / bear / high-volatility" regimes are cleanest to interpret.
We also run everything on **MSFT** and **AAPL** for comparison.

## 2. What is an HMM? (beginner intro)

**One sentence:** a string of invisible "hidden states" hops around over time
according to probabilities, and each state "emits" something we *can* observe. We
only see the emissions and must reverse-engineer the states.

```
hidden:   S₁ → S₂ → S₃ → S₄ → ...      (the market regime — we can't see it)
           ↓    ↓    ↓    ↓
observed: O₁   O₂   O₃   O₄  ...        (daily return & volatility — we measure these)
```

In our project:
- **Hidden state** = the market regime (calm bull, choppy, crisis, …).
- **Observation** = a 2-D vector each day: `[log return, realized volatility]`.

### The two assumptions that make HMMs work

1. **Markov assumption** — *tomorrow's state depends only on today's state*, not on
   the full history. ("If today is a crisis, the chance tomorrow is still a crisis
   is high — I don't need last month to know that.")
2. **Observation independence** — *today's observation depends only on today's
   state*. ("In a crisis regime, today's return is drawn from a wide, near-zero-mean
   Gaussian, regardless of yesterday.")

### The three things that fully define an HMM

| Symbol | Meaning | In our code (`hmmlearn`) |
|---|---|---|
| **π** | probability of each state on day 1 | `startprob_` |
| **A** | transition matrix: P(go from state *i* to *j*) | `transmat_` |
| **B** | emission: how each state generates an observation | Gaussian → `means_`, `covars_` |

Because our observations are continuous (returns/volatility), we use a **Gaussian
HMM**: each regime emits observations from a 2-D Gaussian (a `mean` and a
`covariance`). That's why we can label regimes by looking at `means_` — a state
with a *positive* mean return and *low* volatility is a calm bull market; a state
with *high* volatility is a crisis.

The **diagonal of the transition matrix** (`self_transition`) tells us how "sticky"
a regime is: a value near 1 means the regime persists for many days. Real markets
*do* stay in one mood for a while, and the HMM captures this **persistence**
automatically — this is precisely what a plain non-temporal classifier cannot do.

## 3. The three classic HMM problems

Every HMM textbook poses three questions; our project uses all three.

| Problem | Question | Algorithm | In our code |
|---|---|---|---|
| **1. Learning** | Given only observations, estimate π, A, B | **Baum–Welch (an EM algorithm)** | `model.fit(X)` |
| **2. Evaluation** | How likely is this observation sequence under the model? | **Forward algorithm** | `model.score(X)` → log-likelihood, BIC |
| **3. Decoding** | What is the most likely hidden-state sequence? | **Viterbi** | `model.predict(X)` |

Bonus: `model.predict_proba(X)` (the **forward–backward** algorithm) gives the
*probability* of each state each day, not just the single best guess — we use it for
the "posterior probability" figure.

> **Why we fit with multiple random restarts.** Baum–Welch / EM only guarantees it
> climbs to a *local* optimum, and where it lands depends on the random starting
> point. So `fit_hmm` runs several restarts (`n_restarts`) and keeps the one with the
> highest training log-likelihood. This is a standard, important detail of EM.

## 4. Our data and features

**Source:** daily OHLCV (Open/High/Low/Close/Volume) from Yahoo Finance via the
`yfinance` library, **2000-01-01 to present**, for SPY, MSFT, AAPL. Downloaded by
`make_dataset.py` into `dataset/` (not committed — see `.gitignore`).

From the raw prices we compute three features (`data_loader.py`):

| Feature | Formula | Why |
|---|---|---|
| `log_return` | `ln(price_t / price_{t-1})` | daily % change; log form is additive and more Gaussian (HMM emissions are Gaussian) |
| `realized_vol` | 20-day rolling std of log returns × √252 | how "agitated" the market is lately; needed to separate calm vs crisis regimes |
| `volume_z` | 60-day rolling z-score of log volume | relative trading activity (optional 3rd feature, tested in experiments) |

The model's observation vector is `[log_return, realized_vol]` by default.

**Train/test split (chronological, never random):**
- **Train:** 2000-01-01 → 2021-12-31 (5,477 trading days)
- **Test (held-out):** 2022-01-01 → present (1,110 trading days)

We split by time, not randomly, because shuffling time-series data would let the
model "peek into the future" — a classic and serious mistake.

## 5. How we built the project, step by step

This is the actual path we followed, so you can reproduce or extend it.

1. **Scaffold the model.** `data_loader.py` (load + features + split) and
   `hmm_model.py` (fit with restarts, BIC model selection, regime summary).
2. **Get the data.** Wrote `make_dataset.py` to pull SPY/MSFT/AAPL from Yahoo into
   `dataset/`.
3. **Choose the training window.** We deliberately train on **2000–2021** and test
   on **2022+**. (We later *measured* whether a longer window helps — see
   experiment (c).)
4. **Fit and select K.** `select_n_states` sweeps K ∈ {2,3,4,5,6,7,8}, fits each
   with multiple restarts, and picks the K with the lowest **BIC** (which trades off
   fit quality against model complexity). For SPY the BIC curve reaches a clear
   interior minimum at **K = 7** (it turns back up at K = 8), so 7 regimes is a
   genuine choice, not an artifact of where we cut off the search.
5. **Evaluate.** Compute the **held-out per-observation log-likelihood** on 2022+
   data — the model's ability to explain data it never saw.
6. **Interpret.** `regime_summary` reports each state's mean return, mean volatility,
   and self-transition probability, which we use to name the regimes.
7. **Visualize.** `visualize.py` produces the report figures.
8. **Run controlled experiments.** `experiments.py` measures how performance changes
   as we vary the covariance type, the feature set, the training window, and the
   model class (HMM vs. non-temporal baselines), plus a cross-ticker comparison.
9. **Review & test.** A separate reviewer pass checked correctness (state/color
   consistency, no look-ahead leakage, consistent per-observation LL across HMM vs.
   `sklearn` baselines) and re-ran everything end-to-end. A `pytest` suite
   (`test_pipeline.py`, 19 tests) locks in the feature formulas, the chronological
   split, the parameter counting, and the HMM probability properties.

> **A note on the K search range.** Our first pass searched only K ∈ {2,3,4,5} and
> BIC picked K = 5 — but the BIC curve was still falling at the boundary, meaning K
> was pinned to the edge rather than genuinely optimal. We then widened the search
> to {2,…,8} and found a true interior minimum at **K = 7**. This is the project's
> clearest example of "measure how performance changes when you change the model,"
> and a reminder to always confirm a selection curve bottoms out inside its range.

## 6. How to run it

```bash
# 1. install dependencies
pip install -r requirements.txt        # numpy, pandas, hmmlearn, matplotlib, scikit-learn, pytest
pip install yfinance                    # only needed to download data

# 2. download data into dataset/  (SPY, MSFT, AAPL; 2000 -> today)
python make_dataset.py
#   custom:  python make_dataset.py SPY --start 1990-01-01

# 3. fit the model + print the BIC table, held-out LL, and regime summary
python hmm_model.py dataset/SPY.csv

# 4. generate all report figures into figures/
python visualize.py dataset/SPY.csv

# 5. run all experiments -> tables into outputs/ (+ one figure)
python experiments.py

# (optional) run the unit-test suite (fast, no network/data needed)
pytest -q
```

Outputs:
- `figures/` — 6 PNGs (see below)
- `outputs/` — 5 Markdown result tables (experiments a–e)

> You may see `Model is not converging` lines while fitting — these are harmless
> tiny EM oscillations from `hmmlearn`; every selected model still returns a finite
> log-likelihood and `converged=True`. They are silenced inside `experiments.py`.

## 7. Results: the regimes we found

On SPY, BIC selects **K = 7** (held-out per-observation log-likelihood **5.92**).
Sorting the states by mean return gives a clean, interpretable spectrum from deep
crises up to calm bull markets — no labels were ever provided, the model discovered
this on its own. (`regime_summary` also attaches a heuristic `label` per state:
*crisis / neutral-choppy / bull*, derived from its mean return and volatility.)

| Regime (by ↑ mean return) | Auto-label | % of days | Mean daily return | Annualized volatility | Self-transition |
|---|---|---|---|---|---|
| Deep crisis    | crisis         | 4%  | −0.037% | 0.54 (very high) | 0.974 |
| Stress         | crisis         | 9%  | −0.010% | 0.29 | 0.951 |
| Mild down      | crisis         | 16% | −0.010% | 0.17 | 0.913 |
| Choppy         | neutral/choppy | 14% | +0.008% | 0.21 | 0.933 |
| Normal up      | bull           | 16% | +0.038% | 0.13 | 0.902 |
| Steady bull    | bull           | 22% | +0.047% | 0.10 | 0.931 |
| Calm bull      | bull           | 19% | +0.079% | 0.071 (very low) | 0.961 |

Three things to notice — each one validates the theory:

- **Volatility and return move in opposite directions** across the regimes: the
  calm states drift up, the turbulent states fall. The HMM rediscovered this
  well-known market pattern with **no supervision**.
- **Every self-transition is > 0.90**, confirming regimes are *persistent* — the
  Markov chain is capturing the real-world stickiness of market moods.
- **The deep-crisis regime is rare (4%)**, which is exactly why having a long
  training history matters (it needs to see 2008 and 2020 to learn that state).

> *Why 7 and not 5?* On the original short search range {2,3,4,5}, BIC was still
> decreasing at the boundary, so K was pinned to the edge — an unreliable choice.
> Widening the range to {2,…,8} revealed the true interior minimum at K = 7. This
> is a good lesson: always confirm a model-selection curve actually *bottoms out*
> inside your search range.

## 8. Experiments and findings

All experiments report **held-out per-observation log-likelihood** on the 2022+
test set (higher = better generalization). Tables are saved in `outputs/`.

### (a) Covariance type — `outputs/exp_a_covariance_type.md`
At the selected K=7: `full` wins on both held-out LL (5.922 vs `diag`'s 5.894) and
BIC (−63,996 vs −63,524) — at this model size the within-regime return–volatility
correlation carries enough signal to justify the extra parameters. `tied` (5.390)
is clearly worse; `spherical` (5.853) trails too. On timing (`fit_seconds`): `full`
is slowest (~9.4s), `spherical` fastest (~4.0s), `diag` in between (~7.2s).
**Takeaway:** regimes genuinely differ in their covariance structure, and at K=7
the full covariance earns its cost. (Fun fact: at a smaller K=4, `diag` used to
win — the best covariance complexity depends on the number of states.)

### (b) Feature ablation — `outputs/exp_b_feature_ablation.md`
`[log_return]` alone vs `+realized_vol` vs `+volume_z`. Adding **realized
volatility** transforms the model (it's what lets it tell "calm" apart from
"crisis"); adding **volume_z** does not help here. *(Caveat, also noted in the
table: log-likelihoods across different feature dimensions live in different spaces
and aren't strictly comparable — read it as "does this feature produce a coherent
regime model," not a raw LL ranking.)*

### (c) Training window — `outputs/exp_c_training_window.md` + `figures/06_training_window.png`
Sweeping the training start year {2000, 2005, 2010, 2015} against the same 2022+
test set, held-out LL is **best with the longest history** (2000 → 5.922); every
shorter window is worse (5.875–5.897). **Takeaway:** more history wins here
— the extra data (including the 2008 crisis) outweighs the non-stationarity
concern. This directly answers the design question "is older data too old?": for
this model, no.

### (d) HMM vs. non-temporal baselines — `outputs/exp_d_baseline.md`
The key experiment. We compare the HMM to models with **no temporal structure**:

| Model | Held-out per-obs LL |
|---|---|
| **HMM (K=7)** | **5.922** |
| GMM (K=7, same emissions, no transitions) | 4.605 |
| Single Gaussian | 4.106 |

The HMM beats the same-K GMM by **+1.32 per observation**. The *only* difference
between the HMM and the GMM is the transition matrix (the Markov structure), so this
gap is direct evidence that **modeling regime persistence over time genuinely
helps** — the whole premise of the project.

### (e) Cross-ticker — `outputs/exp_e_cross_ticker.md`
Running the full pipeline (K-sweep 2…8) on each ticker: SPY and MSFT select
**K = 7**, AAPL selects **K = 8**. Held-out per-obs LL: **SPY 5.92 > AAPL 4.91 >
MSFT 4.87**. SPY (the broad index) is by far the most predictable, consistent with
single stocks carrying extra idiosyncratic noise. The table also reports the full
selection time (`fit_seconds`, ~26–32s per ticker for the whole 7-value sweep with
5 restarts each). *(Caveat: AAPL's BIC is still lowest at the K = 8 boundary, so its
true optimum may be even higher — for AAPL specifically the search range could be
extended further.)*

### Summary of findings
- The HMM's **temporal structure is the main source of its advantage** (d).
- **Volatility is the essential feature** (b); volume adds nothing here.
- **Full covariance earns its extra parameters at K=7** — best held-out LL and BIC (a).
- **More training history helps** on this data (c).
- The approach **generalizes across tickers**, working best on the broad index (e).

## 10. How this maps to the CS179 syllabus

This project is a natural meeting point of several lectures:

- **Markov chains** (lec 06) — the hidden state's transition matrix.
- **Likelihood / MLE** (lec 02b) — `score`, BIC, model selection.
- **Latent-variable models & EM** (lec 10) — Baum–Welch *is* EM; the restarts
  address EM's local-optima behavior.
- **Inference / variable elimination** (lec 05) — the forward and Viterbi
  algorithms are dynamic-programming inference on the HMM chain.
- **Structure / model selection** (lec 09) — choosing K by BIC.

It satisfies the project rubric: it uses a probabilistic graphical model (not a
plain classifier), has a quantitative metric (held-out log-likelihood), uses
sufficient data, and **measures how performance changes as the model and data
change** (experiments a–e).
